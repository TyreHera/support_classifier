# О проекте

**Классификатор обращений для автоматизации службы поддержки** — это интеллектуальный сервис, разработанный на основе методов машинного обучения и обработки естественного языка (NLP).

Основная задача проекта заключается в автоматической обработке текстовых обращений пользователей и определении:

* категории обращения;
* уровня приоритета заявки.

Разработанное решение позволяет сократить время первичной обработки обращений и снизить нагрузку на сотрудников службы поддержки.

Проект выполнен в рамках кейса **«Классификатор обращений для автоматизации службы поддержки»**.

---

# Функциональные возможности

Сервис предоставляет следующие возможности:

✅ Приём текстовых обращений пользователей;

✅ Очистка и предобработка текста;

✅ Лемматизация русскоязычного текста;

✅ Удаление стоп-слов;

✅ Векторизация текста методом TF-IDF;

✅ Автоматическая классификация обращений;

✅ Определение приоритета заявки;

✅ REST API для взаимодействия с моделью;

✅ Веб-интерфейс для демонстрации работы сервиса.

---

# Архитектура проекта

Система состоит из трёх основных компонентов:

### 1. Модуль обучения модели

Отвечает за:

* загрузку датасета;
* подготовку данных;
* обучение модели;
* оценку качества;
* сохранение обученной модели.

### 2. API-сервис

Реализован на FastAPI.

Предоставляет HTTP-интерфейс для получения предсказаний от обученной модели.

### 3. Пользовательский интерфейс

Реализован с помощью Streamlit.

Позволяет пользователю вводить текст обращения и получать результат классификации через браузер.

# Принцип работы

Работа сервиса включает следующие этапы:

### 1. Предобработка текста

Входной текст:

```text
У меня не проходит оплата банковской картой
```

Преобразуется следующим образом:

```text
проходить оплата банковский карта
```

На этапе обработки выполняются:

* приведение к нижнему регистру;
* удаление специальных символов;
* удаление стоп-слов;
* лемматизация.

### 2. Векторизация

Подготовленный текст преобразуется в числовой вектор с использованием TF-IDF.

### 3. Классификация

Модель логистической регрессии определяет наиболее вероятную категорию обращения и уровень приоритета.

### 4. Возврат результата

Пользователь получает результат в формате JSON.

# Установка и запуск

## 1. Клонирование репозитория

```bash
git clone https://github.com/username/support-ticket-classifier.git

cd support-ticket-classifier
```

---

## 2. Создание виртуального окружения

Windows:

```bash
python -m venv venv

venv\Scripts\activate
```

Linux / macOS:

```bash
python3 -m venv venv

source venv/bin/activate
```

---

## 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

---

## 4. Обучение модели

Перед запуском сервиса необходимо обучить модель.

```bash
python обучение_модели.py
```

После успешного завершения будет создан файл:

```text
model_combined.joblib
```

---

## 5. Запуск API

```bash
uvicorn веб_сервис:app --reload
```

После запуска API будет доступен по адресу:

```text
http://127.0.0.1:8000
```

Документация Swagger:

```text
http://127.0.0.1:8000/docs
```

---

## 6. Запуск пользовательского интерфейса

В отдельном терминале выполните:

```bash
streamlit run локальное_приложение.py
```

После запуска откроется браузер с интерфейсом приложения.

# Оценка качества модели

Для оценки качества используется метрика Accuracy.

Модель обучается на разделённых данных:

* 80% — обучающая выборка;
* 20% — тестовая выборка.

Дополнительно используется:

* балансировка классов (`class_weight='balanced'`);
* униграммы и биграммы (`ngram_range=(1,2)`).
